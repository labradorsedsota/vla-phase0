## Buggy App 采集结果（负样本）

### Bug 1: db.commit() 遗漏

**影响用例：** TC-L1.1（创建旅行持久化）

**Golden 行为：** 创建旅行 → 刷新 → 旅行仍存在
**Buggy 行为：** 创建旅行 → 302 重定向后旅行不出现

**验证方式：** curl POST /trip/create → 数据未持久化
```
POST /trip/create → 302 FOUND (redirect)
GET / → 旅行列表中不存在刚创建的旅行
```

**根因：** `create_trip()` 函数执行了 `db.execute(INSERT ...)` 但缺少 `db.commit()`，事务在连接关闭时回滚。

---

### Bug 2: 排序方向反

**影响用例：** TC-L1.4（账单列表展示）

**Golden 行为：** 记录按日期 DESC（最新在前）
**Buggy 行为：** 记录按日期 ASC（最旧在前）

**mano-cua session：** sess-20260401163705
**采集证据：**
| 排列位置 | Golden App | Buggy App |
|---------|-----------|-----------|
| 第1条（上） | 住宿 4/3 ¥600 | 餐饮 4/1 ¥300 |
| 第2条（中） | 交通 4/2 ¥150 | 交通 4/2 ¥150 |
| 第3条（下） | 餐饮 4/1 ¥300 | 住宿 4/3 ¥600 |

**根因：** SQL 查询使用 `ORDER BY date ASC, id ASC` 而非 `DESC`。

---

### Bug 3: 结算方向反

**影响用例：** TC-L1.5/L1.6（差额计算+结算方案）

**Golden 行为：** 差额 = paid - owed（实付 - 应承担）
**Buggy 行为：** 差额 = owed - paid（应承担 - 实付）

**mano-cua session：** sess-20260401163824
**采集证据（3人3笔消费 共¥1050）：**
| 人员 | Golden 差额 | Buggy 差额 | 说明 |
|------|-----------|-----------|------|
| 小明(付¥300) | -¥50 | +¥50 | 方向反 |
| 小红(付¥150) | -¥200 | +¥200 | 方向反 |
| 小李(付¥600) | +¥250 | -¥250 | 方向反 |

**结算方案对比：**
| Golden | Buggy |
|--------|-------|
| 小明 → 小李 ¥50 | 小李 → 小明 ¥50 |
| 小红 → 小李 ¥200 | 小李 → 小红 ¥200 |

**根因：** `balance = {m: owed[m] - paid[m] for m in members}` 应为 `paid[m] - owed[m]`。
